Intro

Vue components create reusable chunks of code for constructing user interfaces.

Components can be registered with
Vue.component('my-component-name', { /* ... */ })

Props stands for "properties" and refers to information passed into a component .

A component may include a setup function setup(){..} which sets up the component with life-cycle hooks, computed properties and other facilities belonging to that component.

In the following example, a setup function concludes with a return statement exposing a variable missionList for use by its containing component.

setup() {

  const missionList = [

    { id: 1, text: 'Write plain HTML fallback page' },

    { id: 2, text: 'Create unit tests' },

    { id: 3, text: 'Add style' },

    { id: 4, text: 'Draft app docs' }

  ]

  return {missionList}

}

See HTML example .

Note in the basic component example, the return from the setup function exposes this variable to the Vue instance.

Vue allows components to hold reactive values (or state) inside each individual invocation of that component. This state can record the changing condition of a component, eg counting button clicks or user input.

Vue.component('button-counter', {

  setup(){

  let count = value(0)

  return {count}

  },

  template: `

    <button v-on:click="count++">

      You clicked {{ count }} times.

    </button>

  `

})

See HTML example .
